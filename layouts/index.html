§01 — Why

A network you can reason about.

Most modern CNIs reach for eBPF and bespoke overlays — a datapath you debug with a decompiler.

Routewright takes the other road. It composes the routing, filtering, and load-balancing subsystems the Linux kernel has shipped for decades, and speaks BGP to your fabric the way a router does.

The result is a cluster network you can inspect with the tools already in your muscle memory — ip route, nft, ipvsadm, tcpdump, gobgp. A pod prefix is just another route in the RIB. A Service is just an IPVS virtual server. When something breaks at 3 a.m., the path is visible — not compiled away.

Embedded GoBGP v4

One speaker per node with BFD-driven sub-second ECMP failover. TCP-MD5 authenticated, per-family sessions.

§02 — How it works

The node owns its network identity — before Kubernetes exists.

A per-node daemon is the sole BGP speaker. It brings up the management IP over BGP, programs an ECMP default into the FIB, and pins kubelet's --node-ip — so the control plane has zero blast radius against node reachability.

§03 — Load-bearing invariants

The decisions that keep nodes reachable.

The management plane is immutable at runtime. The cluster can add peers and advertisements — it can never take a node off the network.

Immutable mgmt plane
node.yaml owns the management address. CRDs are strictly additive — never overriding it. SIGHUP reloads additive fields only.
BGP-driven failover
A peer going down sends no per-route withdrawal, so the watch loop prunes a downed peer's next-hops from the FIB. Measured ~1s with BFD.
Per-family sessions
Each session negotiates the AFI of its transport family. IPv6 NLRI over a v4 session yields a next-hop the kernel rejects — so dual-stack peers per family.
BFD on · GR off
On uplinks they conflict in GoBGP's in-process model. Planned stops drain via RFC 8326 GRACEFUL_SHUTDOWN instead.
Authentication
TCP-MD5 on every session. GoBGP has no TCP-AO, so MD5 is the contract with the fabric.

§04 — Status

Early, and honest about it.

Milestone M0a — the per-node BGP host daemon — is complete and proven end-to-end against real gobgpd top-of-rack switches across IPv4, IPv6 and dual-stack. The CNI proper — controller, agent, IPAM, services — is in progress.

Routewright is built for people who would rather operate a routing domain than trust a black box. If that's you, the roadmap is worth a read.
